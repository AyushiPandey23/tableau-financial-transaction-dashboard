# Transaction Data Analysis Dashboard (Tableau)

## Problem Statement

This project focuses on analyzing transaction data to understand user behavior, transaction patterns, and payment trends.  
The dashboard helps in identifying transaction distribution across different cities, payment methods, and merchants.  

It also enables analysis based on age group, device type, currency, and purpose of transactions, providing a comprehensive view of financial activity.

---

## Steps Followed

- Step 1 : Loaded dataset into Tableau  
- Step 2 : Performed data cleaning and preparation  

- Step 3 : Created calculated field:
  - Age Group (for grouping customers based on age)  

- Step 4 : Designed interactive dashboard using multiple visuals  

- Step 5 : Created visuals:
  - Card visual for Number of Transactions by Age Group  
  - Column chart for Total Transactions by Payment Method and Merchant  
  - Map visual for City-wise transaction analysis  

- Step 6 : Added filters:
  - Bank Name Received  
  - Device Type  
  - Payment Mode  
  - Currency  
  - Purpose  

- Step 7 : Applied filters to all charts for consistent data interaction  

- Step 8 : Enabled chart interactivity where visuals act as filters for each other

## Insights

### [1] Transaction Distribution
Transactions were analyzed across cities to identify high-activity regions.

### [2] Payment Analysis
Different payment methods and merchants were compared to understand usage patterns.

### [3] Customer Behavior
Age group analysis helped in understanding transaction behavior of different user segments.

### [4] Filter-Based Insights
Dynamic filters allowed analysis based on device type, currency, and transaction purpose.

### [5] Interactive Analysis
Charts acting as filters enabled deeper and more flexible data exploration.
